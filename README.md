# Jodo: The Professional Equalizer

Jodo is an accessible, agentic operating system designed to bridge the inclusivity gap for professionals. In India's rapidly growing digital economy, talent is often assessed by one's ability to navigate complex, non-accessible web interfaces rather than actual capability. Jodo levels the playing field by functioning as a transparent accessibility middleware layer.

## Core Features & Architecture

The architecture relies on a dual-process system bridging an intelligent backend orchestrator with an accessible frontend interface. 

### 1. Agentic Navigator
An autonomous workflow execution engine that interacts directly with web platforms using the browser's Accessibility Tree (AXTree). Instead of relying on brittle DOM scrapers, Jodo understands the semantic structure of a page, ensuring reliable navigation and form execution across diverse job portals and SaaS tools.

### 2. Cognitive Insight Engine (CoT Introspection)
To build trust, Jodo moves away from opaque processing. Every action generated by the system is paired with a verifiable "Logic Trace." This trace (Observation -> Reasoning -> Decision -> Action) is broadcast in real-time to the user via WebSockets, ensuring radical transparency. Users are always aware of exactly what the system is doing and why.

### 3. Predictive Career Pathing (Lag-Llama Engine)
A time-series forecasting module that analyzes professional trends to suggest the optimal time for engagement. By processing historical job posting data and response rates, the engine forecasts windows of high probability for application success, maximizing the user's return on effort.

### 4. Multimodal Inclusivity Modules
Jodo is built specifically for users with diverse needs:
- **Jodo-Sign**: Real-time translation of interface elements into sign language vector animations for Deaf/Hard of Hearing users.
- **Screen-Reader Optimization**: The Shadow DOM overlays are inherently designed with ARIA compliance, ensuring seamless interaction with native screen readers.

## Daily Utility in the Indian Market

For Indian professionals, the digital divide is not just about internet access; it is about interface accessibility. Job portals, government compliance sites, and enterprise software often lack basic accessibility standards. 

Jodo operates as a silent partner. A user can specify their intent—such as applying for a specific role or submitting a compliance form—and Jodo will navigate the interface, interpret the requirements, and execute the steps. By forecasting optimal engagement times and providing real-time sign language translations, Jodo ensures that talent and skill remain the primary metrics for professional advancement, regardless of physical or cognitive accessibility barriers.

## Getting Started

### Prerequisites
- Python 3.11+
- Node.js 18+ (for frontend and landing page)
- Google Chrome

### Running the System
1. **Backend**: Start the FastAPI orchestrator.
   ```bash
   cd backend
   pip install -r requirements.txt
   python server.py
   ```
2. **Frontend Extension**: Load the `extension/` directory into Chrome as an unpacked extension.
3. **Landing Page**: View the project portal.
   ```bash
   cd landing
   npm install
   npm run dev
   ```

## Repository Structure

The codebase is split into three main architectural pillars:

### `/backend`
The Python-based intelligent orchestrator.
- `server.py`: The FastAPI server managing WebSocket connections, CORS security, and rate limiting.
- `agent.py`: The LangGraph state machine driving the autonomous agent workflow.
- `models.py`: Pydantic schemas enforcing strict structured JSON outputs from the LLM.
- `ax_parser.py`: Utility for compressing raw HTML/AXTree data into token-efficient strings for the LLM.
- `forecaster.py`: Time-series module (Lag-Llama integration) for predicting optimal job application windows.

### `/extension`
The Chrome Extension that bridges the browser and the backend.
- `content.js`: Injects the Shadow DOM UI, scrapes the webpage's DOM, and streams it to the backend via WebSockets.
- `overlay.css`: WCAG AAA compliant styling for the agentic trace overlay.
- `manifest.json`: Chrome extension permissions and entry points.

### `/landing`
The public Next.js marketing and documentation portal.
- `src/app/page.tsx`: The main landing page, built with Tailwind CSS and strict semantic HTML for perfect accessibility.
- `next.config.ts`: Contains hardened Content-Security-Policy (CSP) headers to prevent XSS.
